Components

  • API provides REST interface. Currently there's only one object type to work with - City.
    Please note that when you open REST API in web-browser, so-called "Browsable API" is used.
    When you add format suffix (.json, .xml) and/or open URL with curl, you get response in specified format (json by default), e.g.:
    /api/city/Kyiv/.xml
    /api/city/Kyiv/.json
    /api/city/Kyiv/
    /api/city/ provides access to basic operations on City objects: create, update, remove, list
    City names are used as Primary Key, so city// returns details about City object with given name.
  • Admin provides simple UI to manage Users and Cities.
    Account: user/password
  • API Docs shows short description/demo for each REST endpoint available
Technologies used: Python, Django, Django REST Framework, REST Framework Swagger, MySQL, Twitter Bootstrap.
